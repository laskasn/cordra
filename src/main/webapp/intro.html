A few things to note before you get started...

Administration

You can administer this Cordra instance using username 'admin' and the password you set in the repoInit.json file before startup. The admin account can also be associated with a public/private keypair. Refer to the manual or the Cordra website for configuring the keypair.

When signed in as admin, an 'Admin' dropdown will appear at the top of the page.

The options of the Admin dropdown allow you to define types (schemas and business rules), declare authentication and access control policies, configure handle record details, etc.

You can authenticate as 'admin' and create user and group records as needed. The create button is right next to the search bar.

By default Cordra is configured to store documents and associate metadata with those documents.

Access to Digital Objects

Depending on the default access controls and controls that are set in the individual objects, the admin and other users may need to authenticate prior to searching, retrieving, or creating objects.

By default, only authenticated users can create digital objects, and all users can search, view, and retrieve those objects.

Web Interface

You can authenticate by clicking on the "Sign in" menu button.

You can search by typing keywords in the text box at the top-center and clicking the "Search" button.

You can click on "All Objects" menu item to view all objects in the system. When first instantiated, the system has no objects to display.

APIs

Cordra is largely meant to be used as well as configured via its APIs. Cordra supports REST as well as DOIP APIs. Refer to the documentation for more details.
